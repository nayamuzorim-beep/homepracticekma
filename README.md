# homepracticekma

이 저장소에 포함된 주요 파일과, 외부에서 접근 가능한 URL 템플릿 및 간단한 설명입니다.

주의: 아래 URL 템플릿에서 `<OWNER>`와 `<REPO>`는 실제 GitHub 소유자와 저장소 이름으로 바꿔 사용하세요.

## 파일 목록

- `index.html`
	- GitHub 웹 보기: https://github.com/<OWNER>/<REPO>/blob/main/index.html
	- Raw(원본) 파일: https://raw.githubusercontent.com/<OWNER>/<REPO>/main/index.html
	- GitHub Pages(배포 시) 예시: https://<OWNER>.github.io/<REPO>/
	- 설명: 생성형 AI 소개용 단일 페이지 HTML입니다. 스타일과 콘텐츠가 포함되어 있어 브라우저에서 바로 열람 가능합니다.

- `.github/workflows/static.yml`
	- GitHub 웹 보기: https://github.com/<OWNER>/<REPO>/blob/main/.github/workflows/static.yml
	- Raw(원본) 파일: https://raw.githubusercontent.com/<OWNER>/<REPO>/main/.github/workflows/static.yml
	- 설명: GitHub Pages로 정적 컨텐츠를 배포하는 Actions 워크플로우 설정 파일입니다.

- `README.md`
	- GitHub 웹 보기: https://github.com/<OWNER>/<REPO>/blob/main/README.md
	- Raw(원본) 파일: https://raw.githubusercontent.com/<OWNER>/<REPO>/main/README.md
	- 설명: 현재 파일로, 저장소의 간단한 설명과 외부 접근 URL 템플릿을 포함합니다.

## 사용 방법

1. 위의 URL 템플릿에서 `<OWNER>`와 `<REPO>`를 실제 값으로 교체합니다.
2. GitHub Pages를 활성화하면 `index.html`은 `https://<OWNER>.github.io/<REPO>/`에서 바로 확인할 수 있습니다.

필요하시면 제가 현재 저장소의 원격(origin) URL을 찾아 템플릿을 실제 링크로 바꿔드리겠습니다.
